# eat_what : 基于用户画像与问答引导的“今天吃什么”智能推荐微信小程序设计

## 主要功能界面
- 登录页面
    - 登录页面提供用户名+密码登录 
    - 同时该页面提供一个注册按钮
    - 注册按钮进入注册页面（输入用户名和密码），保存到DB即可完成注册

- 界面模块设计
    - 底部呈现多个页面进入选项
        1. 首页面（今天吃什么页面）
        1. 个人历史记录页面
        1. 个人主页页面

- 首页面
    - 呈现一个按钮（显示“看看今天吃什么”），点击这个按钮后呈现一个问答表单
    - 问答表单会让用户记录其当前状态信息（问答选项前端会配置好）
        - 表单目的是设计一套交互式问答流程，用于捕捉用户的即时性、场景化需求 ：如“今天想吃点清淡的还是重口的？”、“用餐场景是快餐还是正餐？”、“预算大概多少？”等
            - todo ：这里补充可以考虑的问题
        - 动态问答逻辑：根据前面问题的回答动态调整后续问题
            - 例如：如果用户说“今天想吃清淡”，后续就可以问“要不要推荐汤类？”；如果预算很低，可以减少正餐、推荐快餐或外卖。
            - 权重积累：每次回答一个问题，就给某个标签权重加分（如“辣度+0.8”、“汤类+0.6”）。最终把这些临时权重叠加到用户画像向量中，生成“即时画像”。后续会传递给推荐系统用于推荐
    - 表单提交后，从后台获取相应的推荐菜品数据，形成流式列表反馈给用户
    - 对于呈现的推荐选项，用户可以对推荐菜品选择就决定是你了，后台记录用户选择（可在历史记录查看）并退出回到起始页面
        - 后台还会保存用户当前选择形成的画像，供后面决策进行优化用户偏好： 用户即时画像（那一刻的问答结果），推荐列表中的前N项（正负样本数据），时间戳（方便未来训练模型）

- 历史记录页 ：用户可以回顾历史选择，并可以根据历史选择之前选择的菜品

- 个人主页页面
    - 提供跳转按钮1 ：前往编辑用户信息的页面
        - 录入并管理个人静态信息
            - todo ：这里确认好具体的信息参数（如年龄、性别、身高、体重、健康目标：减脂/增肌/健康饮食、饮食禁忌/过敏原），动态偏好（如口味偏好：酸/甜/苦/辣、菜系偏好）
    - 提交商店菜品信息的按钮，会进入一个表单
        - 需要指定菜品名字，菜系，口味，商店位置等一系列信息，提交成功后，后台根据相应信息生成菜品的权重向量
        - 文本（菜名、口味描述） → 用 NLP 模型生成向量（例如 sentence-transformers）;
        - 数值（价格、热量） → 归一化后拼接到文本向量后面;
        - 标签（菜系、口味） → One-hot / Embedding 编码

## 推荐系统
- 根据用户上传数据生成用户当前的用户画像向量
- 从数据库获取到与用户同城的菜品，并计算出每个菜品的推荐得分（如，距离用户距离（500min内，1km，3km...），菜品向量与用户向量之间的match值）
- 最终返回推荐列表数据

- 用户画像模块设计： 用户可以录入并管理个人静态信息（如年龄、性别、身高、体重、健康目标：减脂/增肌/健康饮食、饮食禁忌/过敏原）和动态偏好（如口味偏好：酸/甜/苦/辣、菜系偏好）。 问答引导模块设计： 设计一套交互式问答流程，用于捕捉用户的即时性、场景化需求（如“今天想吃点清淡的还是重口的？”、“用餐场景是快餐还是正餐？”、“预算大概多少？”等）。 智能推荐模块设计： 核心功能，根据用户画像和问答结果，从食谱/餐厅数据库中筛选并排序，生成推荐列表。 食谱/餐厅信息管理模块： 建立一个包含菜品名称、图片、主要食材、营养成分（卡路里、蛋白质等）、口味、价格区间等信息的数据库。


## 数据库数据
    - 用户个人信息的保存
    - 用户历史选择信息的保存
    - 菜品数据信息/菜品向量
